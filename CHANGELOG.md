# PPBears SaaS 更新日誌

本檔案將記錄此專案所有值得注意的更新與變動。

## [v0.3.2] - 2026-03-27
### 🐛 Hotfix
- **確保全線 www 請求一致性**：修正 `createWooProduct` 內部多餘的正則表達式，確保新建立的客製化商品連結 (permalink) 維持完整的 `https://www.ppbears.com`，避免被截斷為非 www 版本。

## [v0.3.1] - 2026-03-27
### 🔄 部署優化
- **快速開單建置修復**：修正快速開單 (`QuickOrder`) 生成商品連結的去字頭邏輯，並確保新版設定生效。

## [v0.3.0] - 2026-03-27
### 🚀 功能與修正
- **Hostinger 防火牆穿透（最終修正）**：針對 Hostinger `Imunify360` 對於 Render 雲端環境連線 non-www HTTPS (`https://ppbears.com`, Port 443) 會發生 `ETIMEDOUT` 封鎖的問題。現在系統後端的 WooCommerce 服務（包含 `woocommerce.service.ts`, `quick-order.service.ts`, `product.service.ts`）會自動將 API 呼叫的 `baseUrl` 正規化加入 `www.`（如 `https://www.ppbears.com`），徹底繞過 WAF 的阻擋，使「連線測試」、「快速開單」、「商品同步」等功能在生產環境重新正常運作。
- **連線診斷工具升級**：為 Admin Panel 的「測試 WooCommerce 連線」增加 DNS 解析、Render 對外連線 IP、以及三種網路協定變體（non-www HTTPS, www HTTPS, non-www HTTP）的多重連線探測回報，大幅增加雲端除錯能力。

## [v0.2.7] - 2026-03-27
### 🐛 Hotfix
- **WooCommerce 連線根本修正**：新增 `woo-request.ts` 工具函式，改用 Node.js 原生 `node:https` 模組（替代 Node 18 內建 `undici`/`fetch`）進行所有 WooCommerce API 請求。根本原因：Render 雲端環境中 `undici` 使用的 Happy Eyeballs 演算法被 Hostinger/Imunify360 防火牆在 TCP 層級阻擋，導致 `AggregateError`，而原生 `https` 模組使用不同的連線機制可成功穿透。

## [v0.2.6] - 2026-03-27
### 🐛 Hotfix
- **WooCommerce API 連線修正**：為所有對 WooCommerce 的 API 請求（包含連線測試、訂單查詢、快速開單與商品同步）加入標準 `User-Agent` 標頭，解決雲端部署後因 Node.js 預設 User-Agent 被防火牆（WAF）阻擋導致的 `fetch failed` 錯誤。

## [v0.2.5] - 2026-03-27

### ⚡ 快速開單系統 (Quick Order via LINE)
- **指令自動建單**：新增在 LINE 對話中輸入「密碼 開單 姓名 金額」自動在 WooCommerce 複製指定基準商品，建立專屬客戶的新商品。
- **後台自訂設定**：支援於 Admin Panel「系統設定」自訂觸發密碼、基準商品 ID 及回覆訊息模板。
- **動態專屬連結**：自動生成專屬客戶名稱的商品連結與 WooCommerce 商品 ID，並直接回傳給客戶方便下單。

## [v0.2.4] - 2026-03-26

### 🛡️ AI 控制與優先權強化
- **AI 優先規則 (Strict Rules)**：新增「AI 優先規則」設定頁面（位於側邊欄），管理員可設定 AI 絕對不可違背的鐵則（如禁止報價、強制詢問型號等）。
- **指令注入等級最高**：此優先規則會被包裝為 `[ABSOLUTE RULES]` 並注入在 System Prompt 的**最頂層**，優先級高於系統設定、產品資料與知識庫。
- **聊天測試平台**：新增 `/chat-test` 頁面，方便管理員直接在後台模擬客戶對話，驗證 AI Prompt 及知識庫的實際回覆結果。

### 📱 後台行動裝置體驗 (Mobile UX)
- **行動版選單優化**：移除不必要的漢堡選單按鈕，將介面極簡化。現在只需在手機上點擊上方的「網頁標題（例：🏠 儀表板）」，即可滑出側邊欄選單。
- **表格橫向捲動**：所有資料表格在手機版皆支援 `overflow-x: auto`，不再被強制截斷或擠壓。

### 🎧 客服接管功能升級
- **🔒 永久接管模式**：針對進行中的對話，除原有的 24 小時暫時接管外，新增「永久接管」按鈕。啟用後，AI 將被永久停用（`expires_at` 設為 2099 年），直到管理員手動點選「還給 AI」為止，避免重要對話因過期而意外讓 AI 介入。


## [v0.2.2] - 2026-03-26

### 資安強化
- **強制 JWT Secret**：移除不安全的預設值，伺服器啟動時若未設定 `JWT_SECRET` 環境變數，立即中斷並拋出錯誤訊息。
- **密碼升級 bcrypt**：將管理員密碼雜湊從 SHA-256 升級為 bcrypt (cost=12)，並支援向下相容自動遷移（下次登入時自動升級，無需重設密碼）。
- **LINE Webhook 強制簽名驗證**：移除條件性驗證邏輯，`channelSecret` 或簽名 Header 任一缺失時一律拒絕。
- **CORS 來源限制**：改為只允許 `ADMIN_PANEL_URL` 環境變數指定的網域，多一層防禦，未設定時 log 安全警告。

### AI 產品搜尋優化
- **手機型號意圖偵測**：新增手機型號正則判斷，客戶只輸入「17 PRO」、「iPhone 16」、「Galaxy S25」等即可自動觸發產品搜尋，無需帶「手機殼」等購物關鍵字。
- **產品索引注入 AI**：搜尋到的產品資料改為注入 AI 系統提示（不再直接回傳），讓 AI 以自然語言作答，並確保回傳的連結來自最新同步的 `product_index`。

### 重複訊息修復
- **Footer 重複修復**：修復 AI 回覆中「ppbears AI客服」footer 重複出現兩次的問題，改為程式碼追加前先判斷 AI 是否已自動在回應中包含。

## [v0.2.1] - 2026-03-26

### 產品同步與白名單優化
- **遞迴子分類同步**：修復 WooCommerce 分類同步限制，系統現在會自動尋找並遞迴同步指定分類下的所有「子子分類」商品（例如：型號 > Apple > iPhone 16）。
- **同步結果真實呈現**：解除原本 Supabase 的 200 筆查詢上限，並改由 `sync_jobs` 提取系統真實處理的商品數量，精準顯示於介面的綠色橫幅中。
- **白名單模式修復**：修復舊版全站同步殘留問題。現在啟用白名單模式同步時，系統會先清空現有索引，確保資料庫與白名單 100% 同步一致。
- **刪除機制防呆重構**：移除不可靠的瀏覽器 `confirm()` 對話框與發生 CORS 錯誤的 HTTP DELETE 方法。改採安全且支援跨域的 POST 機制，並新增直覺的「確定／取消」行內按鈕防呆設計。
- **商品表格排序功能**：新增欄位點擊排序功能，支援按「商品名稱」、「分類」、「支援機型」、「價格」與「上次同步時間」進行升降冪排序。

## [v0.2.0] - 2026-03-26
- **知識庫文件線上編輯**：新增在管理後台直接點擊並修改舊有與新建立純文字文件的功能，系統會自動儲存並建置語意向量。
- **LINE 載入中動畫**：整合 LINE Messaging API 的載入動畫端點，在系統思考中顯示「等待旋轉圖示」，提升客戶耐心與體驗。
- **AI 自動快捷選單 (Quick Replies)**：自動解析 AI 產生的條列式文字選項，並即時轉換為 LINE 聊天室底部可點擊的原生快捷按鈕。

### 修正項目
- **訂單查詢流程優化**：
  - 修復訂單查詢意圖偵測，現在直接輸入「訂單 133495」格式即可觸發。
  - 修復訂單驗證邏輯，改為「輸入編號 → 系統反問姓名/電話 → 驗證通過顯示結果」。
  - 強化回覆格式，加入物流單號、配送方式、取貨門市、付款狀態等欄位。
  - 修正 WooCommerce API 搜尋參數錯誤。
- **系統穩定性**：補齊 `pdf-parse` 的 TypeScript 宣告檔案，修復 Render 雲端環境無法成功建置的問題。

## [v0.1.0] - 2026-03-26

### 新增功能

- 透過注入 Schema JSON-LD 解決 SEO 結構問題。
